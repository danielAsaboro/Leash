<Warning>
  Mobile is in **early development**. Today **iPhone** does on-device chat, on-device voice, and real private-mesh membership with shared task sync. **Android** runs chat only for now. Tool execution, full RAG over your desktop notes, the proactive heartbeat, and the economy ledger still live on desktop Leash.
</Warning>

The mobile app (`apps/mobile`) is a React Native + Expo project that imports `@qvac/sdk` directly and runs models **natively on the phone**. Text chat works fully on-device after the first download; when you pair to your private mesh, the phone can also sync tasks and borrow a stronger provider's text or vision model over the encrypted mesh.

## How it works

<Steps>
  <Step title="Pick and warm a phone-sized chat model">
    The app ships a curated mobile roster and loads one at a time: **Qwen3 0.6B**, **Qwen3 1.7B**,
    **Qwen3 4B**, or **Llama 3.2 1B**. The default is Qwen3 1.7B.
  </Step>
  <Step title="Run locally first">
    Local turns use `loadModel({ modelType: "llm", modelConfig: { device: "gpu", ctx_size: 4096 } })`
    and stream token-by-token in the same on-device chat store the voice call uses.
  </Step>
  <Step title="Borrow a stronger provider when paired">
    If your phone is joined to a private mesh, it can offload a turn to a provider device over the
    encrypted forward worklet instead of loading that model locally.
  </Step>
  <Step title="Sync tasks on iPhone">
    The iPhone mesh worklet runs a real Corestore + Autobase + Hyperbee task replica, so TODOs stay
    in sync with your Macs without a cloud middleman.
  </Step>
</Steps>

| Setting | Value |
|---------|-------|
| Chat models | Qwen3 0.6B / 1.7B / 4B, Llama 3.2 1B |
| Default model | Qwen3 1.7B |
| Context size | `4096` tokens |
| Device | `gpu` |
| Voice | Whisper EN Small + Supertonic EN (F1) |
| Runtime | `@qvac/sdk` Expo integration |

<Warning>
  A **physical device is required** — llamacpp does not run on the iOS simulator or the Android emulator. Run it with `npx expo start` and open on a real phone.
</Warning>

<Note>
  The iOS app runs the **JSC** JavaScript engine, not Hermes: `@qvac/sdk`'s React Native path ships
  syntax Hermes 0.81.5 can't compile, so flipping it to Hermes red-screens the app at launch. This is
  pinned on purpose — don't revert it. See [Mobile platform](/platforms/mobile) for the full reason
  and [iOS install](/install/ios) for the native build recipe.
</Note>

## Mesh membership (iPhone)

The iPhone is a **real member of your private mesh**, not a thin client. It runs a Corestore + multi-writer Autobase + Hyperbee + Hyperswarm + blind-pairing stack inside a Bare worklet, so it replicates the same encrypted **task** CRDT as your Macs. Pair it from the **Mesh** tab by scanning an invite QR your desktop mints; tap the mesh card to see peers and this phone's models, or **Leave mesh** to drop out. See [The mesh and membership](/explanation/mesh-and-membership) for the model and [Mesh](/platforms/mesh) for the operations.
