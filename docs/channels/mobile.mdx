<Warning>
  Mobile is in **early development**. Today the **iPhone** does on-device text chat **and** joins your private mesh as a real member (syncing the task CRDT). The richer features below — memory, tools, voice — are still planned. Android runs chat only for now.
</Warning>

The mobile app (`apps/mobile`) is a React Native + Expo project that imports `@qvac/sdk` directly and runs an LLM **natively on the phone** — no server, and no network at all after the first model download.

## What's available today

<CardGroup cols={3}>
  <Card title="On-device chat" icon="comment">
    Text conversation with **Llama 3.2 1B (Q4_0)**, running on the phone's GPU.
  </Card>
  <Card title="Token streaming" icon="bolt">
    Replies stream token-by-token as the model generates.
  </Card>
  <Card title="Offline" icon="plane">

  </Card>

## How it works

<Steps>
  <Step title="Download on first launch">
    On mount the app calls `downloadAsset({ assetSrc: LLAMA_3_2_1B_INST_Q4_0 })`, showing a live progress bar (~800 MB weights).
  </Step>
  <Step title="Load into memory">
    `loadModel({ modelType: "llm", modelConfig: { device: "gpu", ctx_size: 2048 } })` brings the model up on the GPU.
  </Step>
  <Step title="Chat">
    Each send calls `completion({ stream: true })` and accumulates the token stream into the assistant bubble. On unmount the model is unloaded (weights stay cached).
  </Step>
</Steps>

| Setting | Value |
|---------|-------|
| Model | `LLAMA_3_2_1B_INST_Q4_0` |
| Context size | `2048` tokens |
| Device | `gpu` |
| Download size | ~800 MB |
| Runtime | `@qvac/sdk` Expo integration (native llamacpp) |

<Warning>
  A **physical device is required** — llamacpp does not run on the iOS simulator or the Android emulator. Run it with `npx expo start` and open on a real phone.
</Warning>

<Note>
  The iOS app runs the **JSC** JavaScript engine, not Hermes: `@qvac/sdk`'s React Native path ships
  syntax Hermes 0.81.5 can't compile, so flipping it to Hermes red-screens the app at launch. This is
  pinned on purpose — don't revert it. See [Mobile platform](/platforms/mobile) for the full reason
  and [iOS install](/install/ios) for the native build recipe.
</Note>

## Mesh membership (iPhone)

The iPhone is a **real member of your private mesh**, not a thin client. It runs a Corestore + multi-writer Autobase + Hyperbee + Hyperswarm + blind-pairing stack inside a Bare worklet, so it replicates the same encrypted **task** CRDT as your Macs. Pair it from the **Mesh** tab by scanning an invite QR your desktop mints; tap the mesh card to see peers and this phone's models, or **Leave mesh** to drop out. See [The mesh and membership](/explanation/mesh-and-membership) for the model and [Mesh](/platforms/mesh) for the operations.
