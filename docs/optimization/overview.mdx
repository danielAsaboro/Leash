Leash is optimised around one constraint: the agent must feel capable while every AI call runs through `@qvac/sdk` on local or mesh-owned hardware. The work is not one trick. It is a stack of routing, prompt, queueing, model, memory, mesh, and mobile choices that keep the 4B-class agent inside its budget.

## The operating principle

Small local models fail differently from cloud-scale models. They can call tools, reason, and write, but they are sensitive to overloaded prompts, too many schemas, dependent multi-step chains, cold model loads, and abandoned decodes that keep the GPU busy.

Leash handles those limits in the harness:

| Pressure | Optimisation |
| --- | --- |
| Too many possible skills | Select one likely skill with lexical + semantic routing. |
| Too many tools | Gate by capability, broker many MCP actions, and keep sentinel tools inert. |
| Long conversations | Compact only the model input; keep the full transcript intact. |
| Multi-agent conversations | Expose one relevant worker, give it a bounded handoff, and return to Leash for synthesis. |
| Slow or abandoned decodes | Cancel safely on disconnect and serialize through the broker. |
| Simple or deterministic turns | Answer early or call a narrow registered tool through the chat API. |
| Weak local hardware | Route to local, private mesh, or public mesh by capability and sensitivity. |
| Repeated delegated history | Reuse KV cache only when the prefix is proven identical. |
| Phone limits | Use JSC, curated model choices, smaller contexts, and mesh borrowing. |

## What this section is

This section is the engineering ledger for those choices. It folds together the polished docs, the Claude-Code architecture analysis, the sawdust build log, and the implementation comments.

It deliberately includes negative results: the continuation nudge did not improve dependent chains, the 600M LoRA recall result was 0% in one reported run, and some tempting resilience work was killed when a spike showed the premise was wrong. Those failures are part of the optimisation story because they kept the shipped system simpler.
