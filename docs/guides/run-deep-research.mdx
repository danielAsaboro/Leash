---
title: "How to run deep research"
description: "Start, monitor, cancel, and inspect detached research runs that synthesize web sources on-device."
---

Use this guide when you want the research surface rather than a normal synchronous chat
turn.

## Start a run

There are two common entry points:

- open `/research`, enter a question, and start the run there
- ask for research in chat and follow the returned research link

The retrieval and synthesis models still run locally. What makes this feature different
is that its source gathering is online.

## Watch the run

The research page shows a live view of:

- the current phase
- round count
- source count
- elapsed time
- the graph of query and source expansion

Runs are detached workers, so the page can be reloaded without losing the run record.

## Cancel without corrupting the runtime

Use **Cancel** when you want to stop pursuing the question. The worker exits after the
current model call settles; it does not hard-abort an in-flight generation.

## Read the report

Finished reports render inline and are also stored under `data/leash-research/` as
markdown artifacts.

## Use your own search backend if needed

If you need a more private or more reliable upstream search source, configure
`LEASH_SEARXNG_URL` through Settings or environment.
