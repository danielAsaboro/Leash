---
title: "How to operate Hypha and the broker"
description: "Run, inspect, and reason about the delegated-compute daemon and the serve broker."
---

Use this guide when you want queueing, warm-model overflow, or mesh visibility beyond
the bare local serve.

## Start the broker

```bash
npm run broker
```

The broker listens on `:11436` and forwards to the local serve on `:11435` unless you
override `LEASH_BROKER_UPSTREAM`.

## Start Hypha

```bash
npm run hypha
```

Hypha listens on `:11437` locally and uses `:11438` for LAN pairing.

## Point clients at the broker

Use the broker instead of the serve directly:

```bash
QVAC_OPENAI_URL=http://127.0.0.1:11436/v1 npm run web:dev
```

Without this change, the web app bypasses broker queueing and mesh overflow.

## Inspect broker state

```bash
curl -s http://127.0.0.1:11436/__broker/stats
```

What to look for:

- `served`: total requests completed through the broker
- `aliases`: per-alias queue depth
- `overflow`: shed and availability-routed counts when peer routing happened

## Inspect Hypha state

```bash
curl -s http://127.0.0.1:11437/health
curl -s http://127.0.0.1:11437/peers
```

What to look for:

- peer count
- warm aliases available on peers
- current delegated inflight work

## Enable or disable major Hypha behaviors

Use environment variables when you start the daemon:

```bash
HYPHA_KV_CACHE=0 npm run hypha
HYPHA_FORWARD=1 npm run hypha
HYPHA_REPUTATION=1 npm run hypha
```

Use one flag at a time when you are debugging behavior changes.

## Restart safely after registry lock issues

If model downloads fail with a registry lock error, stop the serve and Hypha first,
finish the download, then start them again. The registry corestore is single-process.
