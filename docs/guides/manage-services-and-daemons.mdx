Use this guide when you need to start, stop, inspect, or reason about the daemons behind
the assistant and publishing surfaces.

## Open the daemon console

Open `/services`. This page is the operator console for the long-running local
processes that support Mycelium.

## Know what each service does

The main cards cover:

- **Model Serve** for all local model inference
- **Broker** for queueing and overflow routing
- **Hypha** for mesh control, pairing, delegation, and settlement
- **MCP server** for in-chat mesh tools
- **Screen Watcher** for local activity sensing
- **Newsroom** for The Understory pipeline
- **Cron** for scheduled local jobs

## Start a service

Use **Start** on the relevant card, then wait for readiness. Some cards surface a true
ready state only after an HTTP health check answers, so `Starting...` can be correct for
several seconds.

If a service never leaves that state, check the log tail first for a port conflict or
startup error.

## Stop or restart safely

Use **Stop** or **Restart** when the runtime is idle.

Be especially careful with the model serve. Mid-generation interruption is treated as a
high-risk path by the current runtime, so do not stop it casually while work is in
flight.

## Use force-stop only for orphaned processes

Most daemon cards expose force-stop or force-restart for externally launched or orphaned
copies. That is an operator recovery path, not the normal way to manage healthy runtime
processes.

## Use logs and status details actively

Each card exposes recent logs and a detail line. Use them to answer questions like:

- is the service actually answering
- is the peer count changing
- is the newsroom progressing
- is the watcher writing fresh activity

## Keep surface responsibilities separate

- Use **Services** to operate daemons.
- Use **Brain** to configure the assistant.
- Use **Settings** to manage machine-local state.
