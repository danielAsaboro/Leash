title: "Run the desktop app"
description: "Run and package the Electron desktop client — offline, on-device chat outside the browser."
---

`apps/desktop` is a standalone Electron chat client: offline, on-device LLM chat through
`@qvac/sdk`, with no browser and no server to point at. This guide runs it in development and
packages a distributable. It is a sibling surface to the web app, not a replacement —
[Leash](/explanation/what-mycelium-is) is the client brand; Mycelium is the engine both sit on.

## Prerequisites

- The monorepo installed from the root (`npm install` at `mycelium/`, in the foreground).
- Model weights warm-cached under `~/.qvac` — the desktop app runs inference itself via the SDK,
  so it needs the GGUF weights present just like the serve does.

## Run in development

From the repo root, run the desktop workspace:

```bash
npm run dev -w @mycelium/desktop
```

That launches `electron-vite dev` with `--no-sandbox`. The window opens with the chat client
bundling the QVAC SDK directly — the first run downloads weights if they aren't cached, and
after that it runs offline.

## Build a distributable

Package for your platform from the same workspace:

```bash
npm run build:mac   -w @mycelium/desktop   # .app / .dmg
npm run build:win   -w @mycelium/desktop   # Windows installer
npm run build:linux -w @mycelium/desktop   # Linux package
```

Each script type-checks (where applicable), bundles with `electron-vite build`, and packages
with `electron-builder`. `npm run build:unpack -w @mycelium/desktop` produces an unpacked
directory build for quick local testing.

<Note>
This is a workspace inside an npm-workspaces monorepo. Let `electron-builder` resolve
dependencies through the workspace — don't add an `install-app-deps` step, which assumes a
self-contained app folder and fights the hoisted `node_modules`.
</Note>

## Verify

- `npm run dev -w @mycelium/desktop` opens the Electron window and answers a chat turn.
- `npm run typecheck -w @mycelium/desktop` passes.
- A `build:*` run emits a packaged artifact under the app's build output.
