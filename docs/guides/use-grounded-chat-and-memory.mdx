---
title: "How to use grounded chat and memory"
description: "Ground answers in local notes and use the memory tools without leaving your machine."
---

Use this guide when Leash is already running and you want chat answers to lean on your
local graph and saved memories.

## Add material to the graph

Put markdown notes in `data/notes/`. For example:

```md
# preferences

I prefer walking meetings before noon.
```

The graph and retrieval path use these files as source material.

## Ask a retrieval question

In chat, ask for something that appears only in your notes, such as:

> What do my notes say about walking meetings?

Expected result: the answer cites the note it used.

## Save a typed memory

Ask Leash to remember a stable preference or fact. For example:

> Remember that I prefer walking meetings before noon.

That writes a typed memory through the `remember` tool rather than relying only on note
retrieval.

## Recall the memory explicitly

Ask:

> What preferences do you have stored for me?

Expected result: Leash uses the `recall` tool and returns the saved memory items.

## Use both retrieval and memory together

For a mixed prompt, ask for a recommendation that depends on both. For example:

> Plan a meeting style that fits my saved preferences and my notes.

This lets you inspect whether the answer cites notes, invokes recall, or does both.

## If answers are ungrounded

- make the question specific enough that the model has to retrieve a source
- confirm your note is under `data/notes/`
- confirm the serve and embedding alias are healthy
- check whether the relevant tool has been disabled in the UI
