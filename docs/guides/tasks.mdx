title: "Tasks & runs"
description: "Track every unit of work — yours, the newsroom's, and the scheduler's."
---

The **Tasks** page (`/tasks`) is a unified work log across everything Leash does on
your behalf. It has three tabs, each focused on a different kind of work.

## Mine — your to-dos

The **Mine** tab shows tasks that have a human owner: things you created, things the
assistant created for you, nightly-growth suggestions, and cron-scheduled items.

**Create a task in chat:**
> add a task: review the quarterly report by Friday

The assistant calls `create_task` and the new task appears here immediately. Tasks can
be in one of four states: `pending`, `in_progress`, `done`, `cancelled`.

**Filter by status or source** using the chip row at the top:
- **Status** — All / pending / in_progress / done / cancelled
- **Source** — All / user / assistant / dream / cron

Click any task to expand it and see its full description and timestamps.

## Newsroom — the pipeline board

The **Newsroom** tab shows every article in the paper's production pipeline. Each row
is a story and its current stage:

`QUEUED → RESEARCHING → RESEARCH_READY → DRAFTING → REVIEW → PUBLISHED`

**Filter by stage, date, section, or origin** to narrow the view. A `PUBLISHED` story
links directly to its article on `/feed`. Mid-pipeline stories show their last-updated
time — a story that hasn't moved in a long time is likely stuck; restart the Newsroom
daemon from [Services](/guides/services) to let it retry.

## Runs — executed work records

The **Runs** tab is a historical log of every discrete piece of work the daemons have
executed: newsroom pipeline ticks and cron job firings.

Each row shows:
- **✓ / ✗** — whether the run succeeded
- **Producer** — `newsroom` or `cron`
- **Run name** — the specific kind of work (e.g. `discovery`, `drafting`, `my-job`)
- **Started at** — timestamp
- **Duration** — seconds, or `…` if still running
- **Detail** — the error message or exit code for failed runs

**Filter by producer or outcome** to find failing jobs. This tab is the first place to
look when a daemon keeps restarting or a cron job silently fails.
