---
title: "How to debug common runtime failures"
description: "Work symptom-first across serve, broker, Hypha, and the local assistant runtime."
---

Use this guide when the local runtime is up but behavior is wrong, stalled, or missing.

## If a turn hangs at zero tokens

Check these first:

- too many tool schemas were offered and the context budget overflowed
- a tools-enabled model was called without tools

What to do:

1. Restart the serve.
2. Retry with the standard toolset.
3. If you recently added tools or MCP servers, remove them and test again.

## If everything hangs after an aborted or timed-out request

Do not abort in-flight generations against the serve. The current runtime treats
mid-decode disconnects as dangerous because they can wedge later generations.

What to do:

1. Stop new traffic to the serve.
2. Restart the serve.
3. If the machine stays wedged, reboot before continuing to debug.

## If vision features fail

The committed runtime treats `qwen3vl` as a known fragile path.

What to do:

1. Confirm the request is non-streaming.
2. Confirm the request does not set `max_tokens`.
3. If image turns still fail, keep the rest of the stack running and treat vision as
   blocked rather than assuming the whole assistant is broken.

## If embeddings return 500s intermittently

The embedding path is sensitive to concurrent requests.

What to do:

1. Serialize your direct embedding calls.
2. Let the app's own indexing path manage concurrency.
3. Retry only after the previous embed call has settled.

## If model downloads fail with a lock error

The registry corestore is single-process.

What to do:

1. Stop the serve.
2. Stop Hypha if it may have touched the registry.
3. Download the model.
4. Start the stopped processes again.

## If the dashboard shows "is the app still running?"

That usually means the Next.js dev server is down or unreachable.

What to do:

1. Restart `npm run web:dev`.
2. Confirm `http://localhost:6801` responds again.

## If pairing cannot find the other device

What to do:

1. Confirm `npm run hypha` is running on both machines.
2. Confirm both devices are on the same LAN.
3. Confirm the peer advertises the model file you expect at the same `~/`-relative path.

## If answers ignore tools or hallucinate missing tools

What to do:

1. Check whether the tool was disabled in the UI.
2. Check custom prompt overrides for references to removed tools.
3. Retry with the default prompt and tool settings.
