## Warm-cache footprint

After the one-time download, everything re-runs offline. The base spike warm-cache
is the 5-model chat/RAG/STT set.

| Model | On-disk size |
|---|---|
| `GTE_LARGE_FP16` (embeddings) | 639 MB |
| `QWEN3_600M_INST_Q4` | 364 MB |
| `LLAMA_3_2_1B` | 737 MB |
| `QWEN3_4B_INST_Q4_K_M` | 2.38 GB |
| `WHISPER_BASE_Q8_0` | 78 MB |
| **5-model warm-cache total** | **~4.2 GB** (the post-spike cache is ~4.0 GB) |

Additional sensing/generation models expand the cache: `SD_V2_1_1B_Q8_0`,
TTS (~252 MB), the FLUX.2 split-layout trio (~5.1 GB), and the Wan T2V split-layout
set (~14.5 GB, dominated by `UMT5_XXL_FP16` at 11.4 GB â€” this set OOM-kills the
M4's `recommendedMaxWorkingSetSize` of 19069 MB). The full QVAC model cache on the
build machine (`~/.qvac`: models + registry + RAG store) is ~4.9 GB.
