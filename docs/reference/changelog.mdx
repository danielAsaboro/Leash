The day-by-day release history of Leash / Mycelium across the QVAC Hackathon build window
(May 31 – June 21, 2026). Everything here runs on-device through `@qvac/sdk` — no cloud AI.
Filter by tag, or read the engineering war-story in `submission/sawdust.md`.

<Update label="June 16" description="Plugins, first-class subagents, mesh-native task sync & the phone as a real mesh member" tags={["Leash", "Skills", "Mesh", "Agent"]}>
  Four landings. A **plugin system** — Claude-Code-compatible bundles (skills + MCP servers +
  subagents) registered *virtually* and quarantined on install, with five install sources
  (folder / upload / GitHub / marketplace / mesh). **First-class subagents** — a user-level
  store and dashboard parallel to skills, standardized on the AI-SDK agent primitives. **Mesh-native
  task sync** — the task list became a replicated CRDT (LWW `MeshTask` + delete tombstones, an
  oldest-active-member leader), with the web app reading/writing tasks through hypha over live SSE
  and a guarded one-time migration of local tasks into the mesh. And **mobile parity**: the iPhone
  joins the mesh as a real CRDT member (Corestore + Autobase + Hyperbee + Hyperswarm in a Bare
  worklet), syncing the task list peer-to-peer, plus a redesigned Mesh tab, QR-invite scanning with
  expiry timers, and 1:1 dashboard parity with an on-device data layer.

  `202674e` plugin system · `c935453` first-class subagents · `20a4888` subagents on AI-SDK
  primitives · `b7b4cde` MeshTask type · `4f04879` LWW task + delete tombstones · `9a1dfec`

<Update label="June 15" description="The proactive assistant + the mcp-cron scheduler" tags={["Leash", "Agent", "Models"]}>
  Leash became **proactive**. The Assistant Kit (`qwen3-4b` chat + a `classifier` triage model +
  `gte-large` + `qwen3vl`), an editable **constitution** (`soul.md` / `goals.md` / `heartbeat.md`)
  folded into every turn, and a cron-driven **heartbeat** that proposes nudges, dedups them on
  `gte-large`, and classifies each into an **auto / notify / ask** tier with a notifications feed.

  The scheduler moved wholesale onto **mcp-cron** (retiring `leash-cron`): a supervised HTTP
  daemon, a typed MCP client, a schedules-store backed by it with idempotent default seeds, and a
  curated agent-facing scheduling MCP group. Plus the **SDK 0.12.1 → 0.13.1** upgrade with the
  vision patch wired into the runtime, chat models defaulting to a **32 768-token** context window,
  natural-language memory recall + TTS-engine config fixes, mcp-cron bundled into the packaged
  desktop app, and a new **Telegram** channel bridge (owner-only, on-device inference).

  `643a724` proactive assistant (kit + constitution + heartbeat + tiers) · `7a31477` mcp-cron spike ·
  `f0cbbbc` supervise mcp-cron · `871125a` typed cron client · `75aa01c` schedules-store on mcp-cron ·
  `3288c77` seed default schedules · `66f1008` scheduler MCP group · `02bf729` retire leash-cron ·
  `cc7eae8` bundle mcp-cron in desktop · `db70dc5` 32768 default context · `78bf0f0` NL recall + TTS config ·
  `7fd5b1a` SDK 0.13.1 + vision patch · `de67def` desktop runtime manifest · `46ffc72` Telegram channel
</Update>

<Update label="June 14" description="Desktop app matures + a standalone tools server" tags={["Leash", "UI", "Skills"]}>
  The desktop app grew up: a **native shell** with per-user isolation, multi-user auth, and
  localhost bind; **real first-run onboarding** (commit-and-leave with a live runtime-download
  page); and an on-demand daemon overlay with a repeatable release pipeline. The built-in tools
  were **extracted into a standalone MCP server**, the bundled skills aligned to the **Agent
  Skills** standard, and the model-download UX hardened (cancel removes the partial weight, failed
  downloads stay retryable, and the chat picker lists only chat models with actionable
  not-loaded errors).

  `2219b80` standalone tools MCP server · `044998a` native shell + multi-user auth · `e29a726`
  built-in skills + Agent Skills format · `46c2aa3` first-run onboarding · `8065863` retryable
  dropped downloads · `87a2499` cancel removes partial weight · `8b6c8aa` chat-only model picker ·
  `c6edbb7` daemon overlay + release pipeline
</Update>
