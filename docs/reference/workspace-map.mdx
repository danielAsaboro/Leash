## Root workspace

| Path | Kind | Notes |
|---|---|---|
| `package.json` | workspace manifest | npm workspaces root; scripts for spike, apps, and smoke tests |
| `qvac.config.base.json` | serve config | committed model alias source of truth |
| `qvac.config.mjs` | serve wrapper | expands `~/` paths at runtime |
| `scripts/` | operational scripts | smoke tests, probes, helpers |
| `spike/` | validated primitives | warm-cache, inference, RAG, P2P, OCR, LoRA, Autobase |
| `data/` | machine-local state | chats, notes, memories, services, catalogs, Hypha state |
| `docs/` | Mintlify site | this docs site only |

## Packages

| Workspace | Role |
|---|---|
| `packages/brain` | built-in agents and built-in skills shared across web, desktop, and mobile |
| `packages/shared` | shared types, logging, audit helpers, KV session support |
| `packages/senses` | embeddings, graph ingest, RAG index, voice and photo sensing |
| `packages/mind` | routing, council logic, critic, tool orchestration |
| `packages/leash-core` | stores, frontmatter, plugins, routing, tool policy, and shared assistant runtime helpers |
| `packages/mesh` | pairing, provider and consumer mesh logic, graph sync, failover |
| `packages/memory` | memory curation, evaluation, training, serve alias handling |
| `packages/db` | Prisma schema and generated database client |

## Apps

| Workspace | Role |
|---|---|
| `apps/web` | Leash web assistant, dashboard, services console, and the local public landing route (`/`) |
| `apps/desktop` | Electron desktop chat client — offline, on-device inference via `@qvac/sdk` |
| `apps/landing` | standalone public marketing/download app for `useleash.xyz` |
| `apps/hypha` | delegated-compute daemon and mesh control plane |
| `apps/hub` | original always-on mesh app |
| `apps/edge-node` | original lightweight consumer app |
| `apps/newsroom` | The Understory pipeline daemon |
| `apps/leash-broker` | queueing and overflow proxy in front of the serve |
| `apps/leash-mcp` | MCP server for mesh pairing tools |
| `apps/leash-tools-mcp` | built-in MCP daemon hosting the capability tool groups on `/mcp/<group>` |
| `apps/leash-watch` | local screen watcher |
| `apps/leash-telegram` | Telegram bridge into the same on-device assistant runtime |

## Data and logs

| Path | Contents |
|---|---|
| `data/notes/` | markdown notes used for retrieval |
| `data/voice/` | voice memo inputs |
| `data/photos/` | image inputs |
| `data/hypha/` | Hypha seed, mesh store, invites, and economy state |
| `data/leash-*.json*` | web-app stores for chats, tasks, memories, prompts, schedules, MCP, and services |
| `apps/*/logs/` | app-local audit logs |
| `spike/logs/` | spike audit logs |

## Docs and evidence boundaries

| Path | Boundary |
|---|---|
| `mycelium/docs/` | product and engineering docs site |
| `submission/` | hackathon evidence, build notes, and public reporting |
| `resources/` | cached external reference material |

<Note>
`mycelium/docs/` is the only markdown-bearing docs area inside the code workspace. Spike
results, evidence summaries, and working logs stay outside it in `submission/`.
</Note>
