## Config files

| File | Purpose |
|---|---|---|
| `qvac.config.base.json` | Machine-neutral source of truth for served model aliases and preload behavior |
| `qvac.config.mjs` | Wrapper that expands `~/` to the current machine's home directory |
| `data/leash-mcp.json` | Stored MCP server rows for the web app |
| `packages/db/prisma/schema.prisma` | Prisma schema for the database package |

## Serve configuration

The committed serve config is `qvac.config.base.json`. Paths inside it use `~/`, and
`qvac.config.mjs` expands them at runtime. Do not create a `qvac.config.json`; the CLI
prefers `.json` and would shadow the wrapper.

| Alias | Model | ctx_size | Role |
|---|---|---|---|
| `qwen3-4b` | `QWEN3_4B_INST_Q4_K_M` | 32768 | chat + agent driver (tools on, dynamic); `default: true` |
| `gte-large` | `GTE_LARGE_FP16` | â€” | embeddings (RAG, effort routing, skill matching) |
| `medpsy` | `MEDGEMMA_4B_IT_Q4_1` | 8192 | health specialist (QVAC MedGemma; tools on, dynamic) |
| `qwen3vl` | `QWEN3VL_2B_MULTIMODAL_Q4_K` + mmproj | 8192 | vision |
| `parakeet` | `PARAKEET_TDT_0_6B_V3_Q8_0` | â€” | speech-to-text |
| `supertonic` | `TTS_EN_SUPERTONIC_Q8_0` | â€” | text-to-speech (`voice: F1`, `ttsSpeed: 1.05`) |

<Note>
The serve's built-in `ctx_size` default is `1024`. The committed chat and health aliases
override it because tool-carrying turns need larger budgets.
</Note>

## Data locations

| Path | Contents |
|---|---|
| `~/.qvac/` | model weights, registry, KV cache, and RAG store |
| `data/` | per-machine stores: chats, tasks, memories, skills, research, secrets, catalogs, and Hypha state |
| `data/notes/` | your markdown notes (RAG source) |
| `data/hypha/` | Hypha seed, mesh store, invite, and economy state |
| `apps/*/logs/` | per-app audit evidence |
