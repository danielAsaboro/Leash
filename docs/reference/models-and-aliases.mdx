## The catalog browser

The **Models** tab surfaces the full SDK catalog — hundreds of entries from `@qvac/ai-sdk-provider` —
on top of the committed aliases.

| Facet / control | Values | Notes |
|---|---|---|
| Status filter | All / Configured / Downloaded / Available | "Available" is the un-downloaded residual |
| Kind filter | Text / Image / Speech / Embedding / OCR / Translation | |
| Hidden families | `gpt`, `bitnet` | hidden from "Available" unless already configured/cached |
| Per-model **Share** | on / off | advertises the alias to the mesh via a daemon deny-set |
| Per-model **Compute** | CPU / GPU / default | `config.use_gpu`; default is GPU on Metal |
| Per-model **Context size** | ≈1024–32768 | live RAM-fit estimate: fits / tight / too-big |
| **Preload** | `true` when added | warms the weight at serve start |

Lifecycle changes (add, compute, context size) apply on the serve's next load of the model, not
instantly. The catalog is dumped to `data/leash-models-catalog.json` by a spawned child script.

## Alias constraints

| Constraint | Detail |
|---|---|
| Chat and health aliases | committed with tools enabled |
| Vision alias | used through the vision route; request shape remains stricter than text models |
| Image generation alias | `LEASH_IMAGE_MODEL` defaults to `sd`, but no `sd` alias is committed in `qvac.config.base.json` |
| Mesh advertising | peers only advertise aliases they can actually serve on that machine |
