**Leash** is a private assistant that runs on your own hardware. It answers from your
notes, files, activity, and past conversations; drives tools with your approval; borrows
compute from your device mesh; and keeps every AI path on-device through `@qvac/sdk`.

Underneath Leash is **Mycelium**, the local-first runtime for models, memory, tools, skills,
agents, and encrypted mesh compute. Leash is what you use. Mycelium is what makes it work.

<Note>
  There is no cloud-AI fallback. Chat, embeddings, RAG, vision, speech,
  delegated inference, and LoRA all route through `@qvac/sdk`.
</Note>

## How the system fits together

```text
clients -> mind -> senses -> memory
             |        |
             +--------+-> encrypted mesh
```

- **Clients** are the surfaces: web, desktop, iOS, Telegram, and background daemons.
- **Mind** is the agent layer: Conductor routing, prompts, tools, skills, sub-agents, approvals.
- **Senses** turns local context into searchable graph data: notes, files, OCR, STT, photos, activity.
- **Memory** curates recall and trains personal LoRA adapters on-device.
- **Mesh** pairs devices, replicates context, delegates inference, and records receipts.

Read the deeper model in [Architecture](/explanation/architecture), or inspect the repo map in
[Workspace map](/reference/workspace-map).

## Evidence for reviewers

The repo includes a structured JSON evidence bundle under `evidence/`. Start with:

| File                                         | What it shows                                                |
| -------------------------------------------- | ------------------------------------------------------------ |
| `evidence/manifest.json`                     | Top-level counts, date windows, and generated evidence files |
| `evidence/chats/{mac-mini,mbp}/index.json`   | Chat counts, message counts, and date ranges by device       |
| `evidence/qvac/{mac-mini,mbp}/summary.json`  | QVAC model, RAG, delegation, and LoRA events                 |
| `evidence/hypha/{mac-mini,mbp}/summary.json` | Mesh pairing, delegation, completion, and adapter events     |
| `evidence/data/{mac-mini,mbp}/summary.json`  | Folded runtime/data evidence by device                       |

For the full submission map, see [Requirement map](/hackathon/submission-artifacts).

## Constraints

- **QVAC only.** Every AI path goes through `@qvac/sdk`.
- **Offline after warm cache.** First run can download model weights and bootstrap the mesh; the
  hot path is designed to run with no network.
- **Real implementation only.** Packages and apps exist because their working code landed.
- **Audit logs stay on.** JSONL and structured JSON evidence are part of the verification story.
