**Leash** is a private assistant that runs on your own hardware. It grounds answers in your
notes, screen activity, and past conversations, drives your machine with your approval,
borrows compute from your other devices, and keeps every inference on-device through
`@qvac/sdk`. There is no cloud AI in the loop.

Under the hood, Leash is the flagship surface of **Mycelium** — a five-layer, local-first
engine (mesh, senses, mind, memory, clients) that hosts the runtime, the encrypted device
mesh, the agent economy, and the nightly memory loop. Leash is what you use; Mycelium is
what makes it work.

## Non-negotiable constraints

- **QVAC only.** All inference, embeddings, RAG, delegated compute, vision, speech, and LoRA
  work go through `@qvac/sdk`. There is no cloud AI fallback.
- **Offline after warm-cache.** First run can download model weights and bootstrap the DHT.
  After that, local chat, retrieval, and mesh operations work without network access.
- **Real runtime code only.** The workspace adds packages and apps only when they land as
  working implementations. The spike, smoke tests, and daemons are all real paths.
- **Audit logs stay on.** Spike and runtime flows emit JSONL audit records and are part of
  the verification story.

## Docs conventions

These docs follow Diataxis on purpose:

- Tutorials teach one happy path.
- How-to guides solve one concrete job.
- Reference pages stay factual and lookup-oriented.
- Explanation pages handle architecture, rationale, and tradeoffs.

Hackathon evidence, spike results, and build logs stay in `submission/`, not in this docs
tree. When a docs page needs to point at evidence, it links out instead of copying the
material.
