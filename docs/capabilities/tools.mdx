All tools live in `leash-tools-mcp` as toggleable groups. Toggle any group in **Brain → MCP** to add or remove its schemas from the active toolset.

## Tool reference

| Tool | Approval | Description |
|------|----------|-------------|
| `search_graph` | No | Semantic search over notes, memories, and screen activity via RAG |
| `recall` | No | Retrieve a specific memory or note by keyword |
| `list_memories` | No | List all stored memories |
| `save_memory` | No | Persist a new memory fact |
| `forget_memory` | No | Delete a specific memory |

### MCP Admin group (skill-gated)

This group is never always-on — it only activates when a skill that explicitly lists `mcp-admin` tools is active. This prevents the 22-schema cap from being consumed by management tools on every turn.

| Tool | Approval | Description |
|------|----------|-------------|
| `install_mcp_repo` | **Yes** | Clone, install, build, and register an MCP server from a GitHub/listing URL |
| `upsert_mcp_server` | **Yes** | Create or update one server entry by hand (http/sse/stdio) |

→ Full reference: [MCP Admin tools](/capabilities/tools/mcp-admin)

## Approval flow

Tools marked **Yes** in the Approval column pause the agent loop and show an approval card in the chat UI with the exact call parameters. You approve or reject; the agent continues either way.

The default approval setting for each tool is configurable in **Brain → Tools**. You can flip any tool to "auto-approve" or "always ask" per your preference.

## Schema cap

The active toolset is capped at ~22 schemas (empirically verified — above this the 4096-token prompt overflows on `qwen3-4b` and the decode hangs). The agent handles this automatically:

- Computer turns activate only the 3 computer tools
- Chat turns activate everything except computer + MCP Admin
- A skill's declared `tools:` override this, capped at 18
