Leash's live suites are root scripts that run against the actual serve, web app, or mesh helpers. They are not package-local `npm run test` commands.

## Prerequisites

```bash
# 1. Start the serve
npm run qvac

# 2. Wait for models to load
curl http://127.0.0.1:11435/v1/models | jq '.data[].id'

# 3. For authenticated chat-route suites, start the web app
npm run web:dev
```

For authenticated chat-route suites, provide a valid dashboard session cookie through `LEASH_COOKIE` or `/tmp/leash-cookie.txt`.

## Serve-backed live suites

```bash
npm run smoke:plugins-live
npm run smoke:orchestration
```

These talk to the live QVAC serve and prove:

- real embedding-based plugin activation
- real specialist inference
- real multi-agent orchestration

## Authenticated chat-route suites

```bash
LEASH_COOKIE='leash_session=...' npm run stress:long-chat-turn
```

This drives the real `/api/leash/chat` surface over 30–50 turns and writes a JSON report under `logs/stress-long-chat-*.json`.

## Mesh and transport checks

```bash
npm run smoke:forward-loopback
npm run smoke:forward-provider
npm run smoke:forward-metering
npm run test:mesh:tasks
npm run test:hypha:tasks
```

Use these when you are checking forward transport, metering, or mesh task sync rather than the chat route.

## Spike proofs

```bash
npm run spike:inference
npm run spike:rag
npm run spike:p2p:consumer
npm run spike:lora
```

Spike runs emit JSONL records under `spike/logs/`.

## Interpreting failures

**`serve is down` errors** — the test couldn't reach `http://127.0.0.1:11435`. Start `npm run qvac`.

**`model not READY` errors** — the alias the test needs isn't in the config or hasn't finished loading. Wait 60s and retry, or add the alias to `qvac.config.base.json`.

**`LEASH_COOKIE` / session errors** — the authenticated chat suites need a valid dashboard session cookie. Export `LEASH_COOKIE` or write `/tmp/leash-cookie.txt`.

**Heartbeat 403s** — `/api/leash/heartbeat` is internal-token gated; use the scheduler, or send `x-leash-internal`.

**Timeout on first assertion** — cold start. The first inference call loads the model or waits behind another request. Re-run once the serve is warm.
