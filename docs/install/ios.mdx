<Warning>
  The iOS app is in early development. It currently supports on-device chat only — no mesh, no memory, no tools. The full feature set lives in the macOS desktop app.
</Warning>

The iOS app runs **Llama 3.2 1B (Q4_0)** entirely on the phone via the `@qvac/sdk` Expo integration — no server, and no network after the first model download.

## Requirements

- iPhone or iPad
- **Physical device required** — `llama.cpp` does not run on the iOS Simulator (native llama bindings). The model load fails on a simulator.
- ~800 MB free storage for the model weight

## What works

- On-device text chat with Llama 3.2 1B
- Token streaming — words appear as they generate
- Offline after the first download (the model is cached on device)

## Not yet available

- Memory and grounding (notes, screen activity, RAG)
- Skills and tool use
- Mesh (connecting to your Mac's Leash instance)
- Voice, research, feed, economy

## Run from source

<Steps>
  <Step title="Install deps">
    ```bash
    npm install   # FOREGROUND only
    ```
  </Step>
  <Step title="Plug in a physical device">
    The Simulator cannot load the model — a real iPhone/iPad is mandatory.
  </Step>
  <Step title="Start Expo">
    ```bash
    cd apps/mobile
    npx expo start        # then open on the device
    # or build natively: npx expo run:ios --device
    ```
  </Step>
</Steps>

## What happens on first launch

The app downloads the model (progress %), then loads it into **GPU** memory with `ctx_size: 2048`. After that, chat is local and offline; subsequent launches reuse the cached weight.

## Connecting to your Mac (planned)

The roadmap includes a "remote Leash" mode where the phone talks to your Mac's Hypha daemon over the mesh — the full feature set without a large local model. That needs the mesh + auth layers not yet exposed on mobile.
