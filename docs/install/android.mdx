<Warning>
  Android support is in early development. It shares the React Native + Expo codebase with iOS but runs **chat only** today — the mesh worklet is built for iOS, so private-mesh membership (which the [iPhone](/install/ios) now has) is not yet wired on Android.
</Warning>

The Android app runs **Llama 3.2 1B (Q4_0)** entirely on the device via the `@qvac/sdk` Expo integration — no server, and no network after the first model download.

## Requirements

- An Android phone or tablet
- **Physical device required** — `llama.cpp` does not run on the Android Emulator (native llama bindings). Use a real device.
- ~800 MB free storage for the model weight

## Run from source

<Steps>
  <Step title="Install deps">
    ```bash
    npm install   # FOREGROUND only
    ```
  </Step>
  <Step title="Plug in a physical device">
    Enable USB debugging; the emulator cannot load the model.
  </Step>
  <Step title="Start Expo">
    ```bash
    cd apps/mobile
    npx expo start        # then open on the device
    # or build natively: npx expo run:android --device
    ```
  </Step>
</Steps>

## What happens on first launch

The app downloads the model (progress %), then loads it into **GPU** memory with `ctx_size: 2048`, and chat streams tokens locally and offline thereafter.

<Note>
  GPU acceleration depends on device vendor support. On some chipsets inference may fall back to CPU — slower, but still functional.
</Note>

## Capabilities

On-device chat and streaming today. Unlike [iOS](/install/ios) — which now joins the private mesh and syncs tasks — Android does not yet have mesh membership (the worklet bundle is iOS-only), nor memory, tools, voice, research, or economy.
