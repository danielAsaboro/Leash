`qvac.config.base.json` is the authoritative serve configuration. It lives at the monorepo root alongside `qvac.config.mjs`, which is the wrapper the QVAC CLI actually loads.

## The two-file pattern

| File | Purpose |
|------|---------|
| `qvac.config.base.json` | Plain JSON, edited by Leash's dashboard. Machine-neutral — use `~/` paths. |
| `qvac.config.mjs` | Wrapper: expands `~/` to the machine's home dir, then re-exports. The CLI loads this. |

**Never recreate `qvac.config.json`** — the CLI prefers `.json` over `.mjs` and it would shadow the wrapper. Leash edits `qvac.config.base.json` directly; the wrapper picks up the changes.

## Schema

```json
{
  "serve": {
    "models": {
      "<alias>": {
        "model": "SDK_CONSTANT_NAME",
        "preload": true,
        "default": true,
        "config": {
          "ctx_size": 32768,
          "use_gpu": true
        }
      }
    }
  }
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `model` | string | SDK catalog constant (e.g. `QWEN3_4B_INST_Q4_K_M`) |
| `src` | string | Raw file path (for custom GGUFs not in the catalog). Mutually exclusive with `model`. |
| `preload` | boolean | Load this model on serve start. Default: `true` |
| `default` | boolean | Use this alias when no alias is specified. Only one alias should be `default: true`. |
| `config.ctx_size` | number | Context window in tokens. Default: `32768` for chat models |
| `config.use_gpu` | boolean | Force GPU (`true`) or CPU (`false`). Omit to let the SDK decide. |

### TTS config

Speech (TTS) models require extra fields:

```json
"supertonic": {
  "model": "TTS_EN_SUPERTONIC_Q8_0",
  "preload": true,
  "config": {
    "ttsEngine": "supertonic",
    "language": "en",
    "voice": "F1",
    "ttsSpeed": 1.05,
    "ttsNumInferenceSteps": 5
  }
}
```

The dashboard's Kit wizard fills these in automatically.

### Vision config

Vision models (`qwen3vl`) need a `projectionModelSrc` pointing at the mmproj GGUF:

```json
"qwen3vl": {
  "model": "QWEN3VL_2B_MULTIMODAL_Q4_K",
  "preload": true,
  "config": {
    "ctx_size": 8192,
    "projectionModelSrc": "~/.qvac/models/<mmproj-cacheFile>.gguf"
  }
}
```

The Kit wizard computes this path from the catalog's `cacheFile` for the projection SKU (`MMPROJ_QWEN3VL_2B_MULTIMODAL_Q4_K`).

## Chat config — the fields that matter

The constant name is not enough: a chat alias needs the right `config` block, or tool calls, reasoning, or loading itself silently break.

### The diffusion-vs-chat naming trap

`QWEN3_4B_Q4_K_M` — the constant **without** `INST` — is a **diffusion** model (`sdcpp-generation`), not an LLM. Pointing a chat alias at it raises `MODEL_SRC_TYPE_MISMATCH`. The real 4B chat LLM is `QWEN3_4B_INST_Q4_K_M`. Always confirm a constant's catalog `engine`/`addon` before serving it as chat.

### Chat config keys

| Key | Where | Value | Why |
|-----|-------|-------|-----|
| `tools` | `config` | `true` | Tool calls are silently never parsed unless this is set |
| `toolsMode` | `config` | `"dynamic"` | Required alongside `tools: true` for the agent to call tools |
| `ctx_size` | `config` | `32768` (`DEFAULT_CTX_SIZE`) | Below ~2048 the tool schemas and Qwen3 chain-of-thought overflow the prompt |
| `reasoning_budget` | **`generationParams`** | `0` | Disables Qwen3 "thinking" so it doesn't burn the context. Lives in `generationParams` on the `completion()` call, **not** in the serve `config` / `modelConfig` |

The shipped chat config is `{ "tools": true, "toolsMode": "dynamic", "ctx_size": 32768 }`. `reasoning_budget` is passed at call time, not in the config file:

```ts
completion({ modelId, history, generationParams: { predict, reasoning_budget: 0 } })
```

See [Chat & reasoning](/models/modality/chat) for the model picture, [Troubleshooting](/help/troubleshooting) for the failure symptoms, and [Benchmarks](/reference/benchmarks) for measured tok/s by size.

## Editing safely

The dashboard writes config via a **promise-mutex** with a fresh read per edit and atomic rename (write-to-tmp then rename). This prevents concurrent dashboard clicks or hand-edits from losing data.

After any config edit, **restart the serve** — there is no live-reload API.

## Environment variable overrides

| Variable | Default | Description |
|----------|---------|-------------|
| `QVAC_OPENAI_URL` | `http://127.0.0.1:11435/v1` | Where the agent calls the serve |
| `QVAC_MODELS_DIR` | `~/.qvac/models` | Model weight cache directory |
| `QVAC_CONFIG_PATH` | `qvac.config.mjs` (at repo root) | Path to the config wrapper the CLI loads |
| `LEASH_DATA_DIR` | `<repo>/data` | Per-user data directory |
