`qvac.config.base.json` is the authoritative serve configuration. It lives at the monorepo root alongside `qvac.config.mjs`, which is the wrapper the QVAC CLI actually loads.

## The two-file pattern

| File | Purpose |
|------|---------|
| `qvac.config.base.json` | Plain JSON, edited by Leash's dashboard. Machine-neutral — use `~/` paths. |
| `qvac.config.mjs` | Wrapper: expands `~/` to the machine's home dir, then re-exports. The CLI loads this. |

**Never recreate `qvac.config.json`** — the CLI prefers `.json` over `.mjs` and it would shadow the wrapper. Leash edits `qvac.config.base.json` directly; the wrapper picks up the changes.

## Schema

```json
{
  "serve": {
    "models": {
      "<alias>": {
        "model": "SDK_CONSTANT_NAME",
        "preload": true,
        "default": true,
        "config": {
          "ctx_size": 32768,
          "use_gpu": true
        }
      }
    }
  }
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `model` | string | SDK catalog constant (e.g. `QWEN3_4B_Q4_K_M`) |
| `src` | string | Raw file path (for custom GGUFs not in the catalog). Mutually exclusive with `model`. |
| `preload` | boolean | Load this model on serve start. Default: `true` |
| `default` | boolean | Use this alias when no alias is specified. Only one alias should be `default: true`. |
| `config.ctx_size` | number | Context window in tokens. Default: `32768` for chat models |
| `config.use_gpu` | boolean | Force GPU (`true`) or CPU (`false`). Omit to let the SDK decide. |

### TTS config

Speech (TTS) models require extra fields:

```json
"supertonic": {
  "model": "SUPERTONIC_TTS_V1",
  "preload": true,
  "config": {
    "ttsEngine": "supertonic",
    "language": "en",
    "voice": "F1",
    "ttsSpeed": 1.05,
    "ttsNumInferenceSteps": 5
  }
}
```

The dashboard's Kit wizard fills these in automatically.

### Vision config

Vision models (`qwen3vl`) need a `projectionModelSrc` pointing at the mmproj GGUF:

```json
"qwen3vl": {
  "model": "QWEN3_VL_2B_Q4_K_M",
  "preload": true,
  "config": {
    "projectionModelSrc": "~/.qvac/models/qwen3vl-mmproj.gguf"
  }
}
```

The Kit wizard computes this path from the catalog's `cacheFile` for the projection SKU.

## Editing safely

The dashboard writes config via a **promise-mutex** with a fresh read per edit and atomic rename (write-to-tmp then rename). This prevents concurrent dashboard clicks or hand-edits from losing data.

After any config edit, **restart the serve** — there is no live-reload API.

## Environment variable overrides

| Variable | Default | Description |
|----------|---------|-------------|
| `QVAC_OPENAI_URL` | `http://127.0.0.1:11435/v1` | Where the agent calls the serve |
| `QVAC_MODELS_DIR` | `~/.qvac/models` | Model weight cache directory |
| `QVAC_CONFIG_PATH` | `qvac.config.mjs` (at repo root) | Path to the config wrapper the CLI loads |
| `LEASH_DATA_DIR` | `<repo>/data` | Per-user data directory |
