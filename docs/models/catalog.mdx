The QVAC SDK ships a large constant catalog of models across seven categories: chat, embedding, transcription, speech (TTS), image, OCR, and translation. Everything downloads as a GGUF file over the QVAC P2P blob network — no central download server, and offline once cached.

<Tip>
For the exact catalog size on your SDK version, run `qvac catalog --json | jq 'length'` — the count tracks the installed `@qvac/sdk`, so this page avoids quoting a number that could drift.
</Tip>

## Browsing

Open **Brain → Models → Catalog** ([http://localhost:6801/brain](http://localhost:6801/brain)). Each row shows:

| Column | What it is |
|--------|------------|
| **Name** | The SDK constant, e.g. `QWEN3_4B_INST_Q4_K_M`. |
| **Category** | chat / embedding / transcription / speech / image / ocr / translation. |
| **Params** | Model size, e.g. `4B`, `1.5B`. |
| **Quantization** | Precision, e.g. `q4_k_m`, `q8_0`. |
| **Expected size** | Download size. |
| **Fit** | `fits` / `tight` / `too-big` for *your* hardware (see [Overview](/models/overview)). |
| **Kit role** | Which role this SKU fills in the recommended Assistant Kit, if any. |

## Filtering

Filter by category or fit verdict, or search by name. The catalog is generated by
`apps/web/scripts/leash-model-catalog.mts` — it spawns a `@qvac/sdk` child process and writes
`data/leash-models-catalog.json`. It refreshes automatically; if you want to force regeneration
from the repo root, run:

```bash
npx tsx apps/web/scripts/leash-model-catalog.mts
```

<Tip>
  Sort by **Fit** and filter to `fits` to see only the models that comfortably run on your machine before you spend bandwidth downloading.
</Tip>

## Downloading a model

<Steps>
  <Step title="Click Download">
    On any catalog entry. The download runs as a **detached child process** and writes live progress to `data/leash-downloads/<name>.json`.
  </Step>
  <Step title="Track it in Tasks → Downloads">
    Watch percentage, bytes, and state. If a download's process dies mid-way, the row turns into an honest error you can restart.
  </Step>
  <Step title="Wire an alias">
    A downloaded weight does nothing until an alias points at it — see [Aliases](/models/aliases), or use the Assistant Kit below.
  </Step>
</Steps>

<Warning>
  The registry corestore is **single-process**. If your serve had to touch the registry (e.g. to fetch blobs during preload), it holds an exclusive lock — concurrent dashboard downloads then fail with *"File descriptor could not be locked."* Stop Model Serve, download, then start it again.
</Warning>

## The Assistant Kit

The **Kit** button (Brain → Models) wires the recommended set of aliases in one atomic edit:

| Role | Alias | Model |
|------|-------|-------|
| Chat | `qwen3-4b` | `QWEN3_4B_INST_Q4_K_M` |
| Embeddings | `embed` | `GTE_LARGE` |
| Classifier | `classifier` | `QWEN3_600M_INST_Q4` |
| Health | `health` | explicit QVAC MedPsy 4B GGUF source |
| Vision | `qwen3vl` | `QWEN3VL_2B_MULTIMODAL_Q4_K` |
| OCR | `ocr` | `OCR_LATIN_RECOGNIZER_1` |
| STT | `parakeet` | `PARAKEET_TDT_0_6B_V3_Q8_0` |
| TTS | `supertonic` | `TTS_EN_SUPERTONIC_Q8_0` |

The Kit validates **every** SKU (each role's weight plus the vision projection/mmproj) against the catalog before writing — all-or-nothing — so a half-resolved kit never produces a broken config. After wiring the Kit, download the weights (individually or all at once) and restart the serve.

## Model cache

Models live under **`~/.qvac/models/`** (flat GGUF files). On machines with limited SSD space, symlink the whole `~/.qvac` to an external drive:

```bash
mv ~/.qvac /Volumes/ExternalSSD && ln -s /Volumes/ExternalSSD ~/.qvac
```

The SDK follows the symlink transparently. The Models page shows **"external SSD · &lt;volume&gt;"** when it detects one, and **"internal SSD"** otherwise.

<Warning>
  **Never copy `~/.qvac` between machines with plain `rsync -a` or `cp`.** macOS rsync strips extended attributes, and corestore stamps a `device-file` xattr on each store; losing it bricks the store with *"Invalid device file, was moved unsafely."* Use `rsync -aX`, or just re-download per machine — it's P2P anyway.
</Warning>
