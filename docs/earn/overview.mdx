Earn is the optional economy layer for delegated inference. It lets a device serve compute to a public mesh, publish a price, stream a completion, and settle the actual work done.

The rule is simple:

- **Local first.** If the device can answer locally, no mesh route is needed.
- **Private mesh second.** Your private mesh is trusted personal infrastructure. Rates exist in the interface, but they are always `0`.
- **Public mesh last.** Public mesh routes are only considered after the sensitivity and confidence filters pass. A public provider can be free or paid.

<Note>
  There are two mesh classes today: one private mesh per user, and up to 15 public meshes. That gives the runtime one private slot plus fifteen public slots without changing the routing shape.
</Note>

## What earns

A provider earns when all of these are true:

- The request is safe to leave the local device and private mesh.
- The selected public provider advertises a non-zero rate.
- The consumer pre-authorizes the maximum spend before streaming starts.
- The provider settles the metered completion after the run closes.

Free public providers still participate in routing. They advertise a `0` rate, skip payment authorization, and return receipts that show the job was free.

## What stays free

Private mesh delegation stays free even though the same economy fields are present. This keeps the UI and routing code consistent:

- Quotes show `0`.
- Rates show `0`.
- Receipts can still be recorded.
- No x402 pre-authorization is opened.
- No settlement transaction is needed.

## The routing ladder

Leash only expands the route when it is allowed to:

```text
local device -> private mesh -> public mesh
```

Public mesh is the market boundary. It is where identity, rate selection, payment rails, reputation, and receipts matter.
