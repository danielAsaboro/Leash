## Leash in one screen

Leash is a private, offline-capable **personal AI assistant harness** that runs entirely on
your own hardware through `@qvac/sdk` — no cloud AI in the loop. It is general-purpose because
it is extensible: tools, skills, MCP servers, specialist delegates, local modalities, and
memory all sit behind one assistant surface. Underneath sits **Mycelium** — the connected
runtime that ties together local models, retrieval, typed memory, and encrypted mesh compute.

It also perceives your world locally: notes, voice, OCR/photos, recent screen activity, and
the proactive feed all become usable context on your own hardware. And it reasons above the
weight class of any one device by placing heavier work on stronger peers across your mesh
instead of collapsing to a cloud dependency.

It is not a single demo trick. It is one assistant riding one runtime across several trust and
economy tiers: on your **private mesh**, your own devices delegate for free; on a **public
mesh**, outside providers can expose compute free or paid. That is the innovation surface:
general-purpose personal AI with encrypted delegation and an optional compute market, all on
the same QVAC-only runtime. See [Leash and Mycelium](/explanation/what-mycelium-is) for the
product/runtime split and [Architecture](/explanation/architecture) for the layers.

<Note>
  **The QVAC-only rule.** Every inference, embedding, RAG retrieval, and
  multimodal call routes through `@qvac/sdk` to a local serve. There are no
  OpenAI/Anthropic/cloud-AI calls anywhere in the system — a single one would
  disqualify the submission. [See the proof →](/hackathon/qvac-only-proof)
</Note>

## Track fit: General Purpose

The hackathon defines four device-class tracks. Leash targets **General Purpose** — retail devices with ≤32 GB RAM (laptops, desktops, workstations, mini-PCs):

- **Primary device:** an Apple-Silicon Mac mini (≤32 GB), running the full stack.
- **Peers:** additional Macs used as delegation and economy peers over the mesh.

The General Purpose track explicitly calls for multi-agent orchestration, multimodal work, advanced RAG, P2P delegation, fine-tuning/LoRA, and privacy-first tooling — which is the exact surface area Leash exercises.

## A built-in Psy-Models angle

The hackathon also runs a **Psy Models** track for meaningful use of QVAC's specialist health models. Leash serves `health` — the dedicated **MedPsy 4B** specialist — as a first-class alias and routes to it automatically on health intent. So the same General Purpose build also demonstrates the Psy-Models capability without a separate project; see [Models and the catalog](/explanation/models-and-the-catalog).

That matters to the story because private health-record RAG is the clearest proof that this is
not just infrastructure. Health is a high-stakes example of why a personal assistant must be
both private and extensible.

## Offline acceptance

After a one-time warm-cache (first run downloads the GGUF weights and bootstraps the mesh, online), everything runs with no network. The acceptance test is literal: **it works in airplane mode**. There's no login, no API key, no cloud dependency on the hot path.

## Where the evidence lives

This repo carries the public proof set. The evidence is intentionally reachable without parent
directories:

- [How we meet the criteria](/hackathon/how-we-meet-the-criteria) — feature-to-criterion mapping.
- [Executive summary](/hackathon/submission-summary) — the one-page overview.
- [Requirement map](/hackathon/submission-artifacts) — the direct requirement-to-evidence map.
- [Prior-work disclosure](/hackathon/prior-work-disclosure) — the explicit judged-work boundary.
- [QVAC-only proof](/hackathon/qvac-only-proof) — every AI path through `@qvac/sdk`.
- [Evidence & reproducibility](/hackathon/evidence-and-reproducibility) — spike gates, audit logs, offline acceptance, and licensing.
- `evidence/manifest.json` — top-level map for the structured evidence bundle.
- `evidence/chats/{mac-mini,mbp}/index.json` — chat counts, message counts, date windows, and hashes.
- `evidence/qvac/{mac-mini,mbp}/summary.json` — QVAC model, RAG, delegation, and LoRA summaries.
- `evidence/hypha/{mac-mini,mbp}/summary.json` — mesh/delegation event summaries.
- `evidence/data/{mac-mini,mbp}/summary.json` — folded runtime data summaries.
- `spike/logs/*.jsonl` — representative JSONL audit records for inference, RAG, P2P, LoRA, and mesh.
- `evidence/remote-api-calls.json` — structured outbound-call disclosure.
- `evidence/medpsy-demo.jsonl` — committed MedPsy health-record RAG audit run.
