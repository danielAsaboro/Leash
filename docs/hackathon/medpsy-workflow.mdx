The Psy Models track asks for **strong use of a specialized health model**, not a keyword
model-swap. MedPsy is a genuine clinical-adjacent workflow built on the **MedPsy 4B** specialist
(the runtime exposes it under the `health` alias) — and it runs entirely on-device through `@qvac/sdk`, with **zero cloud calls**.

## What it does

A **retrieve-then-read** RAG consult specialized for personal health:

1. **Ingest** your private health records — notes, lab panels, medications, allergies, visit
   summaries — into a dedicated `health-records` RAG workspace (`embed` / `GTE_LARGE_FP16` embeddings, on-device).
2. **Retrieve** the relevant records for the question (the same `search_graph` backend the Mind
   council uses). We retrieve up front rather than waiting for the model to elect a tool call —
   for a health-record domain we *always* want the records, which also makes grounding robust.
3. **Answer** with MedGemma, grounded strictly in the retrieved records, citing each claim as
   `[Source N]` and never inventing a value that isn't in the records.
4. **Verify** the answer's claims against the sources with a second critic pass.
5. **Safety-wrap**: a non-diagnostic clinician disclaimer is *guaranteed* on every answer, and an
   emergency / red-flag question (chest pain, trouble breathing, stroke signs, suicidal thoughts…)
   prepends an urgent-care escalation banner.

Implementation: `packages/mind/src/medpsy.ts` (`runMedPsyConsult`), composed from the
`packages/senses` RAG layer and the council's critic (`packages/mind/src/critic.ts`).

## Run it

```bash
npm run medpsy:demo
```

This ingests the synthetic health-record fixtures in `spike/fixtures/health-records/`, loads
MedPsy + `embed`, runs a grounded consult and an emergency consult, and writes an audit log to
`packages/mind/logs/medpsy-demo.jsonl`. The records are synthetic fixtures; the inference,
embeddings, RAG retrieval, and verification are all real on-device `@qvac/sdk` calls.

## Why this is "strong use"

- The model is a **medical specialist** (MedGemma), not the general chat model with a prompt suffix.
- The answer is **grounded in the user's own records** and **cited**, then **independently
  verified** — the failure mode a single small model can't self-catch.
- Safety is **structural**, not hoped-for: the disclaimer is always present and emergencies escalate.
- It is **private and offline** — the records never leave the device, and after warm-cache the whole
  consult runs in airplane mode.
