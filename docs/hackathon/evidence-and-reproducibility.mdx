## Hardware setup

Leash targets the **General Purpose** track — retail Apple-Silicon devices with ≤32 GB RAM — and
runs as a small mesh of personal Macs, each running its **own single** QVAC serve. This is P2P
delegation between independent devices, not a pooled multi-GPU cluster ([why that distinction
matters](/hackathon/qvac-only-proof)): the mesh passes whole completions between machines, it does
not pool their memory, so each device satisfies its own RAM budget.

### Devices (the personal mesh)

General Purpose targets retail Apple-Silicon inside the **≤32 GB** envelope. The **primary** device
is a **Mac mini (Apple M4, 24 GB)** — comfortably inside the track ceiling. The MacBook Pro is an
**optional delegation peer**, not part of the RAM budget: the mesh passes whole completions between
independent devices and never pools their memory, so each device satisfies its own ≤32 GB budget.

| Field | **GP primary — Mac mini** | Delegation peer — MacBook Pro | Mobile — iPhone 12 Pro Max |
|---|---|---|---|
| Chip | Apple **M4** | Apple M3 Pro | Apple A14 Bionic |
| Unified memory | **24 GB** (≤32 GB ✓) | 36 GB | 6 GB |
| GPU | Apple M4 — **Metal** | Apple M3 Pro — **Metal** | Apple A14 GPU — **Metal** |
| Model identifier | Mac mini (M4) | `Mac15,7` | iPhone13,4 |
| Cores | M4 (capture on device) | 12 (6P + 6E) | A14 (capture on device) |
| macOS / iOS | capture on device | 26.5.1 (25F80) | capture on device |

The mini's GPU/memory ceiling is independently evidenced in
`submission/evidence/tier2c-wan/spike-result.txt` (`recommendedMaxWorkingSetSize = 19069.67 MB`
≈ 75 % of 24 GB unified). The benchmark numbers below were measured on Apple-Silicon Macs with
Metal acceleration. Capture the exact identifier / core count on each device with the command below
and attach the output (the `capture on device` cells are filled from the machine that is travelling
with the operator).

### Storage and the model cache

Model weights live under `~/.qvac/models/` as flat GGUF files; the warm cache is **~4.2–4.9 GB**
across the default model set (per-model sizes on [Benchmarks](/reference/benchmarks)). On
space-constrained machines `~/.qvac` can be symlinked to an external drive — the SDK follows the
symlink with no config change and no reproducibility impact, since a fresh clone still resolves the
default `~/.qvac`. After the one-time warm-cache, that cache is all the system needs to run offline.

## The four spike gates — all GO

The build was gated on a de-risking spike that proved each core primitive end-to-end before any
product work. All four passed.

### A — On-device inference + embeddings

- Warm chat (Llama 3.2 1B): **TTFT 56 ms**, **70.6 tok/s** (cold first-load TTFT 466 ms).
- Embeddings (`gte-large`): a **1024-dim** vector in ~84 ms.
- Device-fit sweep: Qwen3-600M **113.7 tok/s**, Llama 1B **74.7 tok/s**, Qwen3-4B **24.1 tok/s**.

### B — On-device RAG (grounded, cited)

- 3 docs → 7 chunks; retrieval latency **23 ms**; top-chunk relevance **0.768**.
- The grounded answer cited the source; the no-context answer correctly declined — measurably
  different outputs.

### C — Encrypted P2P delegated compute

- Provider registration (cold DHT): **4.57 s**; round-trip **1.27 s**; **TTFT 257 ms**;
  **45.4 tok/s** on the provider's GPU.
- Noise-encrypted transport (Holepunch/Hyperswarm).

### D — On-device LoRA (QVAC Fabric)

- Fine-tune completed in **152.7 s**; train acc **0.86**, val acc **0.85**; **20 MB** adapter.
- Loaded via `modelConfig.lora`; base vs adapter answers differed — behavior changed.

## Structured audit log

Every spike and runtime flow emits **JSONL** audit records (`mycelium/spike/lib/audit-log.ts`),
capturing at least what the hackathon requires — model loads/unloads and per-call performance:
timestamp, source, event, `modelId`, `device`, `prompt`, `tokens`, `ttftMs`, `tokensPerSecond`,
`durationMs`. Records land under `spike/logs/` and per-app `logs/`. Worked examples are in
`submission/evidence/` (e.g. the KV-cache session log and the cross-host settlement records).

## Airplane-mode acceptance

The project treats **"works in airplane mode"** as a release gate. The first run warm-caches
model weights and bootstraps the DHT online; after that, local chat, retrieval, and on-device
mesh operations run with no network. The non-AI remote calls are enumerated in
[QVAC-only proof](/hackathon/qvac-only-proof).
