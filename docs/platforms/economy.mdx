<Warning>
  The economy layer is functional but early. Settlement, metered streaming, and reputation are gated
  behind feature flags and need manual setup. The free delegated-inference path is stable; the
  current dashboard ledger is centered on paid public sessions.
</Warning>

The **Leash economy** is an optional micro-payment layer on top of the mesh. Providers earn for
compute they serve; consumers spend to borrow inference from peers. Everything below is grounded in
`apps/hypha/src/config.ts` ‚Äî the `HYPHA_ECONOMY_*` env knobs.

It is **off by default**: `HYPHA_ECONOMY_ENABLED` defaults to `0`. Set it to `1` (plus a wallet/rail) to opt in.

## Roles

| Term | Definition |
|---|---|
| **Provider** | A device that accepts delegated completions and earns payment |
| **Consumer** | A device that sends delegated completions and pays |
| **Price** | `HYPHA_ECONOMY_PRICE_PER_KTOK` base units per thousand tokens (default `1000`) |
| **Float** | The consumer's pre-authorized spend ceiling per session (`HYPHA_ECONOMY_FLOAT`, default `50000`) |
| **Settlement** | An on-chain transfer of the settlement asset (EVM/Plasma or Solana) |

<Note>
  Units are **base units** of the settlement asset ‚Äî e.g. micro-USDT when `*_ASSET_DECIMALS = 6` (the default).
</Note>

Plasma-first; Solana support is lower-level rail support, not the main submission path.

## Spend limits

The consumer enforces bounded settlement with three ceilings:

| Variable | Default | Description |
|---|---|---|
| `HYPHA_ECONOMY_MAX_PER_TX` | `5000` | Max per single settlement |
| `HYPHA_ECONOMY_MAX_PER_HOUR` | `20000` | Hourly spend ceiling |
| `HYPHA_ECONOMY_MAX_PER_COUNTERPARTY` | `10000` | Per-peer spend ceiling |

## Enable the economy

```bash
# Provider env (.economy.prov.env)
HYPHA_ECONOMY_ENABLED=1
HYPHA_ECONOMY_PRICE_PER_KTOK=500            # base units per 1k tokens
HYPHA_ECONOMY_PLASMA_RPC_URL=http://127.0.0.1:8545
HYPHA_ECONOMY_PLASMA_MNEMONIC="word word word ..."   # quoted!
HYPHA_ECONOMY_PLASMA_ASSET_MINT=0x...
```

```bash
# Consumer env (.economy.cons.env)
HYPHA_ECONOMY_ENABLED=1
HYPHA_ECONOMY_FLOAT=50000                   # pre-auth ceiling per session
HYPHA_ECONOMY_PLASMA_RPC_URL=http://127.0.0.1:8545
HYPHA_ECONOMY_PLASMA_MNEMONIC="word word word ..."   # quoted!
```

## Metered streaming (experimental)

Off by default (`HYPHA_ECONOMY_METERED` = `0`). When on, a session advances its authorization every `HYPHA_ECONOMY_CHUNK_TOKENS` (default `64`) tokens instead of a single pre-auth. If the consumer goes silent past `HYPHA_ECONOMY_ADVANCE_WINDOW_MS` (default `20000`), the provider watchdog force-settles the authorized cap. Optionally (`HYPHA_ECONOMY_REVOKE_ON_CUTOFF` = `1`) the provider also firewall-excludes the stalled consumer for `HYPHA_ECONOMY_REVOKE_TTL_MS` (default `30000`) ‚Äî a non-destructive cooldown, not an unpair.

### The Permit2-witness ladder

Metered settlement is an **escalating authorization ladder**, not one payment per chunk. The policy is pure and lives in `apps/hypha/src/metered.ts`:

- At open, the consumer authorizes a tier-0 cumulative token cap. Before each new chunk it signs a **fresh Permit2 witness** for a strictly higher *cumulative* cap and sends it as an `advance_authorization`. The provider verifies each rung and appends it (`appendRung`), which enforces strictly increasing `tierIndex` **and** `cumulativeTokens` ‚Äî the ladder can only ever escalate, and a replayed rung is idempotent (`applied: false`).
- At close the provider settles **exactly one rung** ‚Äî the highest reached (`rungToSettle`) ‚Äî at `min(reportedTokens, acceptedThroughTokens)`. Money moves **once per session**; on-chain gas is O(1) regardless of how many rungs the consumer climbed. A consumer that authorized rungs 256 then 512 and decoded 512 tokens settles `512`, not the rung sum `768`.

### The idle watchdog (force-settle on abandonment)

If the consumer goes silent past the advance window, the provider's `sweepMeteredIdle` watchdog **force-settles the full authorized cap** (`settleTokensAtCutoff` returns `acceptedThroughTokens` ‚Äî the consumer signed for it, so it is owed) and removes the session. The consumer's next advance then gets `session not found`. This is the abandoned-session backstop: a network-blipped consumer still pays for what it authorized, and the provider never leaves money unsettled. The watchdog state is persisted so the force-settle survives a provider restart.

When `HYPHA_ECONOMY_REVOKE_ON_CUTOFF` = `1`, the watchdog *additionally* drops the stalled consumer from the delegation firewall for `HYPHA_ECONOMY_REVOKE_TTL_MS` ‚Äî a `transientRevoke` cooldown that auto-re-admits, not a forget/unpair (the writer key stays in the mesh).

### Live on-chain evidence

The metered rail is proven end-to-end across two Macs on a local Anvil fork of Plasma (chain `9746`), settling real USDT0 through the x402 `upto` Permit2 proxy. Representative settle transactions (illustrative ‚Äî these hashes ship publicly):

| Tx | What it proves |
|---|---|
| `0x645c56c3` | Highest-rung settle ‚Äî a 2-rung ladder (caps 256‚Üí512), 512 tokens decoded, **512 settled in one tx** (not the 768 rung sum) |
| `0x067348ad` | A 5-rung ladder (tiers 1‚Üí5, caps 128‚Üí384), 360 tokens settled in one tx |
| `0x6f7dcbce` | The idle watchdog firing ‚Äî consumer streamed 128 tokens then went silent 40 s; `sweepMeteredIdle` (8 s window) **force-settled the authorized cap 128** |
| `0xa71a85fd` | An identity-bound provider settle ‚Äî 219 ¬µUSDT0 moved to the provider's bound payee |

Each settle is a single USDT0 `Transfer` payer‚Üípayee. To stand up the local Anvil fork yourself (faucet-free, with the etched `upto` proxy and the fork-clock fix that keeps Permit2's `validAfter` witness fresh), see [Local settlement on Anvil](/help/local-settlement-anvil).

## Verifying the economy

The economy paths are covered by offline smoke and gate scripts (run with `npm run`):

| Script | Covers |
|---|---|
| `smoke:mesh-economy-policy` | private/public mesh caps, zero-rate policy, and paid-session gate |
| `smoke:economy` | ledger derivation and market/accountability view |
| `smoke:receipt-view` | `/economy/receipts` filters, grouping, and pagination |
| `smoke:metered` | The metered ladder arithmetic + state transitions (`scripts/smoke-metered.ts`, pure policy ‚Äî no chain) |
| `smoke:reputation` | Quality-weighted routing tie-break (`effective_cost = pricePerKiloToken / quality`) |
| `smoke:identity` | The wallet‚Üîprovider-key binding + receipt verification |
| `gate:firewall-revocation` | That a provider firewall stop‚Üístart cleanly revokes a consumer without dropping a live link |

<Note>
  Public paid pricing is provider-level today. The public docs should not imply alias-specific
  pricing or equal ledger treatment for zero-rate and paid routes, because the current code does not.
</Note>

## Reputation (experimental)

Off by default (`HYPHA_REPUTATION` = `0`). When on, **paid** delegation targets are tie-broken by `effective_cost = pricePerKiloToken / quality` ‚Äî a peer with a poor quality score is penalized. Free peers stay first. The reputation store and `GET /reputation` are always read-only; this flag only changes routing.

## Self-dealing detection

Each device must use a **distinct wallet** so self-dealing (a consumer paying itself) is detectable on-chain. Use different BIP39 mnemonics per device. Two further opt-in tiers harden this: `HYPHA_ECONOMY_IDENTITY_BINDING` = `1` makes a provider advertise a wallet‚Üîprovider-key proof, and `HYPHA_ECONOMY_VERIFY_RECEIPTS` = `1` counts a receipt toward reputation only if its on-chain txHash moved the asset to the provider's bound payee.
