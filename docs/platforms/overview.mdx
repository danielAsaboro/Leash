Leash runs across several surfaces and links them into an encrypted device mesh. Each surface runs locally; the mesh lets them share inference, memory, and an economy without routing through a cloud.

## Surfaces

| Surface | Status | Description |
|---|---|---|
| **macOS Desktop** | Shipped | Electron app — setup wizard, bundled daemons, OS notifications, auto-update |
| **Web (Next.js)** | Shipped | Local browser UI at `http://localhost:6801` — public landing at `/`, authenticated dashboard at `/home`, `/chat`, `/brain`, and the rest |
| **iPhone / iPad** | Early (WIP) | On-device chat and voice, real private-mesh membership, shared task sync, delegated inference |
| **Android** | Early (WIP) | On-device chat only — no mesh worklet, no task sync, no delegated runtime yet |

## The mesh

The **Mycelium mesh** connects your devices over an encrypted Hyperswarm DHT. Once paired, devices can:

- Share model inference — borrow a model already warm on another machine
- Forward non-delegable work (e.g. vision) to a peer's local serve
- Sync shared task state across your private devices
- Participate in the optional agent economy (earn/spend for compute)

→ See [Mesh](/platforms/mesh), [Earn](/earn/overview), and [Economy](/platforms/economy).

## The Hypha daemon

`hypha` is the mesh control plane — DHT membership, peer discovery, inference routing, and (when enabled) payment settlement. It must run on every device that participates in the mesh.

| Port | Surface | Purpose |
|---|---|---|
| `11437` | localhost | Control plane + health/peers (`/health`, `/peers`) |
| `11438` | LAN | Pairing surface (open only in pairing mode) |

Start it with `npm run hypha`. For the complete port map, see [Runtime ports and processes](/reference/runtime-ports-and-processes).
